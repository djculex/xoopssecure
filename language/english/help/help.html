Help: XoopsSecure

Description

The xoopsSecure module is a tool used to check for security issues in your Xoops installation.

Install/uninstall

Install the module through Admin -> System Module -> Modules.

Uninstall is done the same place Admin -> System Module -> Modules.

Features

XoopsSecure is a module used to check for security issues in your Xoops Installation.
The module's actions are alle but one run from admin side of Xoops. Only a cron scan is possible to run from front-end, but only if log in is from admin.

What it does

Scan for missing Index files

Missing index files is if the server is not secured against directory browsing a way for visitors to see all the content
of your folders. An index file giving a "no permission" or "error 404" will prevent this.

This scan will create - if you permit it - these index files where missing. Check files and folders for right permissions

Check files and folders permissions

Some files and folders ought to have certaing permissions set. For instance read only or writable
This scan will check these files and folders and if configured set the right permissions

Check files and folders for malware

While this modules will not identify ALL malware out there, it will look for know hacker names, malware patterns and know
bad guys that could suddently be added to your script.

It does NOT have the ability to automaticly remove them and
some results could be coincidences but it will give you a scan log to see what is found at least.

Check Coding Standards

This scan will go through pages in a defined folder to check for issues concerning development. It is not as effective as for instance
phpcodesniffer, but it will find the most obvious sinners agains PSR-12 & Xoops Coding Standards.

Some indications of what maybe should be changed..some not
Same options of deleting, ignoring etc as the other scans are possible.

Check server settings

Looking through the server config, software and frameworks

If a hosted web this could be difficult to change, but in local and your own server this could be helpfull to close a few gaps

This page will also display eventual issues concerning MySql, Xoops and Apache.

Automatic / cron scan

Here you could config automatically scanning for changes in your pages. The result will be sent admin email. If a lot of pages suddently have been modified
and you know you havent done it... it could be a good idea to look it over.

Back-up

This feature is both manual and automatic.
Set the automatic backup to run after every scan but max after certain amount of time to not flood the server with backup files.

You can of course download and delete from the admin section.

In the backup/download tab in admin of the module you can also configure wich github repos to follow. These will display under usefull downloads
showing latest releases so you can keep following them.

The github free api being used allow 60 loadings per minute. During development/testing this could result in an empty page.

Just wait a few minutes to refresh.
